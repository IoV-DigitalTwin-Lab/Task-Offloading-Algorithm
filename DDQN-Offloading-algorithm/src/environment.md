# Vehicular Edge Computing (VEC) Simulation Documentation

## 1. System Overview
This project simulates an **Internet of Vehicles (IoV)** environment designed for **Task Offloading** experiments using Reinforcement Learning (specifically Deep Q-Networks). The simulation models a highway environment with moving vehicles, stationary Road Side Units (RSUs), and computational tasks that need to be processed within strict deadlines.

### Key Components
* **Vehicles:** Mobile nodes with dynamic positions, limited batteries, and CPU resources.
* **RSUs:** Static infrastructure nodes acting as central controllers for offloading decisions.
* **Tasks:** Computational jobs generated by vehicles requiring specific CPU cycles and data transmission.

---

## 2. Environment Architecture (`environment.py`)

### The Map (Highway Model)
* **Dimensions:** A wraparound world of 3000m width (`Config.MAP_WIDTH`) and 1000m height (`Config.MAP_HEIGHT`).
* **RSU Placement:** Three RSUs are placed at intervals (x=500, 1500, 2500) along the center line (y=0).
* **Physics Loop:** The map uses a "wraparound" logic. If a vehicle exits the right side ($x > 3000$), it reappears on the left ($x=0$).

### The Simulation Step
The environment is **episodic** but maintains **continuous physics**.
1.  **`reset()`**:
    * Advances global physics (moves all cars).
    * Selects a random vehicle currently inside an RSU's range.
    * Generates a random task for that vehicle.
    * Returns the state vector relative to the active RSU.
2.  **`step(action)`**:
    * Executes the offloading decision (Local, Neighbor Vehicle, or RSU).
    * Calculates Latency and Energy consumption.
    * Computes the Reward based on QoS and Deadlines.
    * **Handover Risk:** Checks if the vehicle leaves the RSU range during the process.

## 3. Entity Definitions (`entities.py`)

### Vehicle
* **Kinematics:** Possesses $x, y$ coordinates, speed ($m/s$), heading (degrees), and acceleration.
* **Resources:** CPU, Memory, and Battery.
* **Relative Features:** Vehicles normalize their data relative to the RSU they are connecting to (e.g., Distance to RSU, Relative Heading).

### RSU (Road Side Unit)
* Acts as the **Agent**. It observes the state of the network and decides where to route the task.
* Has significantly higher CPU resources than vehicles but limited bandwidth channels.

### Task
* **Attributes:** Size (MB), CPU Requirements (Cycles), Deadline (Seconds), QoS Level (1-3).
* **Origin:** Created by a specific vehicle at a specific timestamp.

## 4. Physics & Networking Logic

### Movement (`_update_physics_global`)
Vehicles move based on a standard kinematic formula:
$$x_{new} = x_{old} + (speed \times \cos(heading) \times \Delta t)$$
$$y_{new} = y_{old} + (speed \times \sin(heading) \times \Delta t)$$

### Networking Model
Bandwidth is dynamic and affected by:
1.  **Base Bandwidth:** Fluctuates using a normal distribution (`Config.BANDWIDTH_VAR`).
2.  **Stability Factor:** Penalizes bandwidth if the sender and receiver are moving in opposite directions (Doppler effect/Signal stability simulation).
    * *Formula:* $EffectiveBW = Bandwidth \times (0.5 + 0.5 \times Stability)$

### Latency Calculation
$$Total Latency = T_{transmission} + T_{processing} + T_{jitter}$$
* **Transmission:** $(TaskSize \times 8) / Bandwidth$
* **Processing:** $CPU\_Required / Available\_CPU$

## 5. RL Interface (State, Action, Reward)

### State Space (Input)
A 1D array containing normalized features:
1.  **Task Info:** [Size, CPU_Req, Deadline, QoS]
2.  **RSU Info:** [CPU_Avail, Mem_Avail, Queue_Len]
3.  **Neighbor Vehicles (k=5):** For each candidate, relative features are stacked:
    * [CPU, Mem, Battery, Speed, Rel_X, Rel_Y, Heading, Accel, Tasks]

### Action Space (Output)
An integer representing the index of the target:
* `0` to `k-1`: Offload to specific neighbor vehicle.
* `k` (`Config.MAX_NEIGHBORS`): Process at RSU.

### Reward Function
A weighted sum of three objectives, multiplied by QoS level:
$$Reward = (w_{lat} \times R_{lat} + w_{ene} \times R_{ene} + w_{dead} \times R_{dead}) \times 10 \times QoS$$
* **Penalties:** Large negative rewards are applied if a task fails or if a vehicle leaves RSU range during transmission (**Handover Failure**).
